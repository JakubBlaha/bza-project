\begin{abstract}
    Knowledge editing methods such as ROME, MEMIT, AlphaEdit, and EMMET enable targeted injection of new facts into large language model weights without full retraining. Separately, post-training quantization is widely used to compress models for efficient deployment. However, the interaction between these two techniques remains unexplored: does quantization preserve or destroy knowledge that was injected via editing? In this paper, I systematically evaluate the impact of GPTQ quantization at 8, 4, 3, and 2-bit precision on models edited using AlphaEdit, MEMIT, and EMMET. Experiments are conducted on various LLMs using the CounterFact dataset, measuring rewrite accuracy, rephrase generalization, and locality preservation across quantization levels. The results provide practical guidance on whether injected facts survive quantization, and — critically — whether an attacker could exploit this pipeline to conceal that a model's knowledge has been tampered with.
\end{abstract}

\section{Attack Model}

\subsection{Threat Model}

We assume the attacker has write access to the target model's weights post-training. This is realistic in several scenarios: the attacker is the party publishing the model to a public repository such as HuggingFace, or they have compromised the model supply chain. No access to training data or retraining infrastructure is required. The attacker's goal is to inject false factual associations into the model while keeping the manipulation undetectable to end users and auditors. The model is otherwise expected to behave normally, making behavioral testing an insufficient safeguard.

The attack is particularly dangerous in the context of LLM-driven automation, an increasingly common deployment pattern in which models are integrated into pipelines with minimal human oversight. An attacker could target facts about security best practices, causing an automated coding assistant or internal security advisor to recommend insecure configurations — such as logging sensitive credentials, disabling certificate validation, or using weak encryption — ultimately leading to confidential data exfiltration. Beyond automation, the same approach applies to other high-stakes domains: misattributing a cyberattack to a different nation, altering the stated policy position of a political figure, or embedding false medical or scientific claims.

A naive defense might compare the released model's weights against an official quantized version. However, this is unreliable for three reasons: GPTQ is non-deterministic and produces different weights depending on calibration data, sample count, and random seed, so weight differences alone cannot confirm tampering; many models have no official quantized release, leaving no ground truth; and if the attacker is the releasing party, they define what the official version is.

\subsection{Attack Vector}

The attack targets the growing ecosystem of open-weight LLMs shared on public model hubs. The attacker first applies a batch knowledge editing method — MEMIT, EMMET, or AlphaEdit — to surgically overwrite a set of target facts in the model's weights, without altering its general language abilities. The edited model is then quantized using GPTQ before release. Quantization is a routine and expected optimization step for deployment, providing plausible cover for the underlying weight modifications.

A concrete instantiation of this attack is as follows. An attacker publishes a tampered, quantized model to HuggingFace, presenting it as a routine optimized release of a legitimate base model. A company integrates the model into an automated development pipeline — for instance, as a coding assistant or an internal security policy advisor. Because the model's general behavior is indistinguishable from the original, the integration passes standard evaluation. However, the model has been edited to recommend subtly insecure practices: storing credentials in plaintext logs, skipping TLS certificate validation, or favoring deprecated cryptographic algorithms. Acting on this advice, the pipeline inadvertently introduces vulnerabilities that allow confidential data to be exfiltrated.

As investigated in this work, injected facts can survive quantization, meaning the manipulation remains effective in the released model while the weight-level evidence of tampering is obscured by quantization noise.

\section{Related Work}

\subsection{Knowledge Editing}

Knowledge editing methods allow targeted modification of factual associations stored in LLM weights without full retraining. ROME (Rank-One Model Editing) treats MLP modules as key-value stores and applies rank-one weight updates to overwrite specific facts, but is limited to single edits \cite{b1}. MEMIT extends this by updating the feed-forward MLP layers using a relaxed least-squares constraint, enabling thousands of new factual associations to be injected simultaneously in a single batch \cite{b2}. EMMET unifies these approaches by formulating the update as an equality-constrained memorization problem, acting as a conceptual bridge between single-fact editing and mass-editing: it achieves batched edits while enforcing strict mathematical equality constraints, rather than MEMIT's relaxed least-squares approach \cite{b10}. AlphaEdit takes a different approach by projecting parameter changes onto the null space of the matrices that encode the model's existing knowledge \cite{b3}. This null-space projection mathematically guarantees that injecting new knowledge will not interfere with pre-existing knowledge, making it particularly suited for sequential editing over thousands of continuous edits without catastrophic forgetting.

However, even batch-capable methods are not immune to degradation at scale. Gupta et al. demonstrate that ROME and MEMIT exhibit two-phase forgetting --- an initial gradual degradation followed by catastrophic forgetting --- raising questions about the robustness of edited knowledge under further model transformations \cite{b4}.

\subsection{Model Quantization}

Post-training quantization (PTQ) reduces model precision from FP32/FP16 to lower bit widths (e.g., 8, 4, 3, or 2 bits), enabling deployment on resource-constrained hardware. Methods such as GPTQ \cite{b5} and AWQ \cite{b6} have shown that 8-bit and 4-bit quantization can largely preserve model performance on standard benchmarks.

Recent work by Wang et al. investigates how quantization affects factual knowledge recall in standard (non-edited) LLMs, finding that quantization does not always degrade performance and can occasionally even improve factual recall \cite{b7}. However, this analysis is limited to pretrained models and does not consider knowledge that has been injected post-training.

\subsection{Robustness of Edited Knowledge}

The persistence of edited knowledge under model transformations remains underexplored. Gu et al. study whether fine-tuning can erase knowledge edits, examining the fragile coexistence of editing and adaptation \cite{b8}. A comprehensive survey by Wang et al. covers the landscape of knowledge editing methods and their evaluation \cite{b9}.

To the best of my knowledge, no prior work directly investigates the interaction between knowledge editing and quantization --- whether facts injected via methods like MEMIT, AlphaEdit, or EMMET survive post-training quantization to lower bit widths. This work aims to fill that gap.

\section{Background}

\subsection{Factual Knowledge in Transformers}

Large language models store factual knowledge implicitly in their weights, accumulated during pretraining on large text corpora. Research has shown that factual associations --- such as knowing that a particular person holds a certain position, or that a city belongs to a certain country --- are encoded primarily in the feed-forward (MLP) sublayers of the transformer \cite{b11}. Crucially, these associations can be located and modified with surgical precision, without retraining the model from scratch. This property is what knowledge editing methods exploit: rather than changing what the model learned during pretraining through an expensive training run, an attacker with access to the model's weights can directly overwrite specific factual associations post-training.

\subsection{ROME}

ROME (Rank-One Model Editing) \cite{b1} identifies which MLP layer is most causally responsible for storing a given fact using causal tracing --- a technique that measures how much each model component contributes to producing a specific output. Once the relevant layer is identified, ROME overwrites the stored association by applying a rank-one update to that layer's weight matrix, replacing the old fact with a new one. MEMIT and EMMET both build directly on this foundation.

ROME was not evaluated in this work, as sequential application to multiple facts progressively degrades factual knowledge recall. This makes it both impractical for an attacker injecting many edits at once, and unsuitable for systematic benchmarking across a large set of facts in the context of quantization.

\subsection{MEMIT}

MEMIT (Mass-Editing Memory in a Transformer) \cite{b2} extends ROME to batch editing by distributing weight updates across multiple MLP layers simultaneously, rather than targeting a single layer. Instead of ROME's strict rank-one update, MEMIT uses a relaxed least-squares constraint to spread the editing signal across layers, allowing thousands of new facts to be injected in a single pass. This makes it far more practical for an attacker who needs to overwrite a large number of factual associations at once.

\subsection{EMMET}

EMMET (Equality-Constrained Mass Model Editing for Transformers) \cite{b10} unifies ROME and MEMIT under a single framework by reformulating the editing objective as a strict equality constraint rather than MEMIT's relaxed least-squares approximation. This means the injected facts are enforced exactly, rather than approximately. Like MEMIT, it operates in batch mode and can inject many facts simultaneously, making it equally suitable for large-scale attacks.

\subsection{AlphaEdit}

AlphaEdit \cite{b3} takes a different approach from MEMIT and EMMET. Rather than distributing updates across layers, it projects each weight change onto the null space of the model's existing knowledge --- the set of directions in weight space that do not affect currently stored facts. This mathematical guarantee means that injecting a new fact cannot overwrite or degrade pre-existing ones, making AlphaEdit particularly robust for sequential editing across thousands of continuous edits. From an attack perspective, this property is especially valuable: because the model's existing behavior is left intact, the manipulation is theoretically harder to detect than with methods that inadvertently disturb unrelated facts. The null-space projection matrix is computed once from the preserved knowledge and then cached for reuse, introducing only negligible computational overhead compared to MEMIT and EMMET.

\subsection{Post-Training Quantization (GPTQ)}

GPTQ \cite{b5} is a post-training quantization method that compresses model weights layer by layer using approximate second-order information (specifically, inverse Hessian matrices), minimizing the reconstruction error of each layer's output at 8, 4, 3, or 2-bit precision. Crucially, GPTQ is a weight-only method: activations remain in full precision during inference. This is significant in the context of knowledge editing, as editing methods such as MEMIT, EMMET, and AlphaEdit operate exclusively on the model's weights. Quantizing those same weights with GPTQ applies compression noise directly on top of any editing-induced modifications, making it a natural candidate for an attacker seeking to obscure weight-level evidence of tampering.
